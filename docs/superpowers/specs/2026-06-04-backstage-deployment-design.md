# Backstage Deployment — Phase 1 Design

## Overview

Deploy Backstage as an Internal Developer Portal (IDP) — "AndresKube Homelab Services" — accessible at `https://platform-apps.andreskube.uk`. Phase 1 focuses on getting Backstage running with PostgreSQL, Keycloak SSO, and a user management workflow.

## Architecture

```
Cloudflare
    │
    ▼
cloudflared ──► backstage.backstage.svc.cluster.local:7007
                    │
                    ├── DB: keycloak-db-rw.postgres.svc / database: backstage
                    │         (managed via CNPG `spec.managed.databases`)
                    │
                    └── Auth: Keycloak OIDC @ keycloak.keycloak.svc.cluster.local:8080
                              Realm: "backstage"
                              Client: "backstage"
```

## Deployment structure

### 1. PostgreSQL database

**File: `applications/postgres/cluster.yaml`** — add to existing keycloak-db Cluster:

```yaml
spec:
  # ... existing config ...
  managed:
    databases:
      - name: backstage
        owner: backstage
    roles:
      - name: backstage
        login: true
        passwordSecret:
          name: backstage-db-app
```

CNPG will auto-generate a Secret `backstage-db-app` in the `postgres` namespace containing `username: backstage` and a random `password`.

### 2. Helm chart wrapper

**Directory:** `applications/backstage/`

**`Chart.yaml`** — Backstage Helm chart from https://backstage.github.io/charts, pinned to a specific version (not `latest`).

**`values.yaml`** key settings:

- **App title:** "AndresKube Homelab Services"
- **Base URL:** `https://platform-apps.andreskube.uk`
- **Database:** PostgreSQL via `keycloak-db-rw.postgres.svc`, database `backstage`
- **Auth:** OIDC provider pointing to Keycloak realm `backstage`
- **Container:** Pinned image tag, non-root security context
- **Ingress:** Disabled (traffic comes via Cloudflare Tunnel → ClusterIP)

**`templates/db-secret-reflect.yaml`** — Reflector annotation to mirror `postgres/backstage-db-app` → `backstage/backstage-db-backstage`

**`templates/backstage-secrets.yaml`** — ExternalSecret for:
- `session-secret` (from Vault: `secrets/data/backstage`)
- `KEYCLOAK_CLIENT_SECRET` (from Vault: `secrets/data/keycloak-backstage-client`)

**`templates/realm-configmap.yaml`** — Keycloak realm JSON for `backstage` realm:
- Client `backstage` with redirect to `https://platform-apps.andreskube.uk/api/auth/oidc/handler/frame`
- Roles: `backstage-admin`, `backstage-user`
- Groups/roles claim mapper

### 3. Backstage access repo

**Repository:** `backstage-homelab-access.git` (new, separate from `argocd-homelab-access.git`)

Same pattern as ArgoCD:
- `chart/` — Helm chart with a Kubernetes Job
- `users.yaml` — Usernames and roles only (no passwords)

The Job:
- Runs `kcadm.sh` against the Keycloak `backstage` realm
- Creates users with a random temporary password
- Sets "update password on first login" required action
- Admin retrieves initial creds out-of-band

**In main repo:** `applications/backstage-access/application.yaml` — ArgoCD Application pointing to the new repo.

### 4. Cloudflare Tunnel

In Cloudflare Zero Trust dashboard:
- Subdomain: `platform-apps`
- Domain: `andreskube.uk`
- Type: HTTP
- URL: `http://backstage.backstage.svc.cluster.local:7007`

### 5. Security

- All container images pinned to specific versions
- Non-root user, read-only root filesystem where supported
- Resource limits set
- Secrets never in git — flow through Vault → External Secrets → K8s Secrets
- No passwords in `users.yaml` — temporary passwords set via Keycloak API

## Secrets flow

```
Vault (KV v2)
├── secrets/data/backstage ───────────────── session-secret
├── secrets/data/keycloak-backstage-client ── client-secret
└── (DB password is auto-generated by CNPG, no Vault needed)
        │
        ▼
ExternalSecret (CRD)
        │
        ▼
K8s Secret (in backstage namespace)
        │
        ▼
Backstage pod (env vars)
```

## GitOps delivery

```
Git push ──► ArgoCD ApplicationSet
              (scans `applications/*`)
                  │
                  ▼
              applications/backstage/Chart.yaml + values.yaml
              applications/backstage/templates/
                  │
                  ▼
              K8s cluster
```

## Phase 1 scope

What's included:
- Backstage deployed and running
- PostgreSQL database via managed CNPG
- Keycloak SSO login via `backstage` realm
- Public access via `platform-apps.andreskube.uk`
- Users manageable via PRs to `backstage-homelab-access.git`

What's deferred to later phases:
- Catalog registration of existing services
- Software Templates (self-service actions)
- CI/CD integration
